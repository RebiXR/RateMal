# RateMal
(in order to use it as it is now: needed: node.js, browser 
open a terminal for frontend and backend directory: npm run dev in both, 
access the application over link in frontend terminal )

About RateMal: 
RateMal is intendend as a web application. This application is intended for people with physical or mental disabilities, therefore the layout and content is kept simple in order to keep it user friendly. It was created with the goal to keep patients who have to train in a "Standing board/ tilt table" entertained and to strenghten community through playing together.
How to use: 
First, the user has to chose one of three lobbies, then the user can start drawing with other users. The user can chose between 8 different colors both accessible for the pen and the sticker option. The user can get inspiration from generated promts for the entire group and for themself individually. The users can participate in a guessing game, where one user gets a prompt to draw and the other users can guess what it is. 

MOSCOW
Muss-Kriterien
Die App muss es dem User ermöglichen, auf einem Canvas frei zu malen. 
Die App muss einen Stift in einer Stärke zur Verfügung stellen.
Die App muss es den Usern ermöglichen, gemeinsam miteinander auf einem Canvas zu malen.
Die App muss eine Echtzeitübertragung zwischen Nutzern bereitstellen, für die Zusammenarbeit, damit die Formen schon beim Malen übertragen werden können.
Die App muss es den Mitspielern ermöglichen, sich in einer gemeinsamen Lobby zu verbinden.
Die App muss eine Option für zufällige und einfache Zeichenvorschläge haben. 
Die App muss Eingaben von Standard Input Geräten wie Maus und Tastatur erkennen können.
Die App muss verschiedene Farbmöglichkeiten in Form einer Palette mit 8 Farben anbieten.
Die App muss eine vorgefertigte Form (Fleck-Form) anbieten, die in eine der 8 Farben gesetzt werden kann.


Soll-Kriterien
Diese Anforderungen sollten möglichst erfüllt werden, sind aber nicht absolut notwendig.
Die App sollte eine Rate Funktion haben, die pro Runde automatisch eine zeichnende Person bestimmt und alle anderen als Ratende zuordnet.
Die App sollte es den Ratenden ermögliche nach Abschluss der Zeichenphase die Zeichnung zu sehen und ihre Antworten einzugeben (Textfeld oder Auswahloptionen)
Die App sollte Rate Antworten von mehreren Spielern gleichzeitig verarbeiten können.
Die App sollte große Bedienelemente für Personen mit motorischen Einschränkungen bieten.
Die App sollte Buttons in Form von Piktogrammen zusätzlich zum Text anzeigen.(kognitive Unterstützung)


Kann-Kriterien
Die App sollte das Einfügen von mindestens fünf einfachen Vorlagen und Grundformen ermöglichen. (motorische Unterstützung)
Die App kann kompatibel sein mit alternativen Eingabemethoden, darunter:
Zeichnen mit Stift auf iPad
Sticker-Auswahl über Tastatur
Die App sollte verschiedene Schwierigkeitslevel für Zeichenvorschläge und Ratevorschläge bereitstellen.
Die App kann es Usern ermöglichen, die fertiggestellten Bilder zu speichern und anschließend anzuschauen.
Die App kann es den Usern ermöglichen, zu einem anderen Zeitpunkt an dem Bild weiterzuarbeiten, das Werk kann allerdings nicht mehr zurückgesetzt werden.

would-Kriterien:
Die App kann automatisch eine Auswahl treffen, wenn eine Spielerin auf einen Prompt nicht reagiert oder keine Assoziation bildet, nachdem andere Spieler*innen die Runde abgeschlossen haben.
Die App kann eine minimalmotorische Steuerung unterstützen, bei der Nutzer*innen Aktionen ausschließlich über zwei große Auswahlfelder oder zwei Tasten ausführen können.
Die App kann  eingegebene Antworten der Rate-Runde auswerten und Punkte vergeben:  Ratende bekommen Punkte für richtige Lösung. Der Zeichnende bekommt Punkte, wenn eine definierte Anzahl an Rateversuchen richtig ist
Die App sollte Undo/ Redo-Funktionen bieten beim Zeichnen.  (motorische Unterstützung)
Die App kann kompatibel sein mit alternativen Eingabemethoden, darunter:
Papier mit Drucksensorplatte (Haptik-Unterstützung)
Sticker-Auswahl per Finger
Sticker-Auswahl via Augensteuerung (Eye Tracking)
Die App kann auf einer Sensorplatte gespielt werden, die automatisch das gezeichnete anderen Mitspielern überträgt
die App sollte eine Funktion Stille-Post haben, mit Zeichnungen, die in einer festgelegten Reihenfolge weitergereicht werden.
Die App sollte sicherstellen, dass jeder Spieler nur den direkt vorherigen Schritt in der Stillen Post-Runde sieht
Die App sollte am Ende der Stillen Post Runde, wenn der letzte Spieler seine Zeichnung abgeschlossen hat, die gesamte Kette sichtbar machen.
Die App sollte einen anpassbaren Schwierigkeitsgrad bieten. (kognitive Unterstützung)
Die App sollte bei der Rate Funktion ein Zeitlimit für Zeichen- und Ratephase haben, nach Ablauf endet die Runde automatisch
Die App sollte erste Unterstützung für unterschiedliche motorische und kognitive Fähigkeiten ausgelegt sein.
Die App kann eine anpassbare Linienstärke anbieten. (erweiterte motorische Unterstützung)
Die App kann Zeichenvorschläge automatisch oder auf Anfrage vorlesen.
Die App kann die Möglichkeit haben, Farben und Formen auf einem Papier zu zeichnen, mit Hilfe einer Kamera zu erkennen und zu digitalisieren.
Die App kann eine Stabilisationsfunktion (Glättung ) beim Zeichnen bereitstellen. (motorische Unterstützung)
Die App kann Regeln und Spielanweisungen in einfacher Sprache wiederholt abspielbar machen.
Die App kann vorgefertigte Bilder anbieten, die mit Stickern und Formen verschönert werden können
